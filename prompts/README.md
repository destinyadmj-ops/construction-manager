# Prompts (Master Hub)

このフォルダは「カスタムエージェント用プロンプト」を分割管理するための土台です。

推奨構成:
- `requirements.md` 要件（業務フロー、入力、出力、制約）
- `glossary.md` 用語集（現場用語・略語・帳票名）
- `schemas.md` 画面/API/データのスキーマ草案
- `tests.md` 受け入れ条件・テスト観点
- `implementation_prompt.md` 実装依頼用の統合プロンプト（テンプレ）

まずは以下を埋めると、以降の実装プロンプトがブレにくくなります。

- `requirements.md`（前提・スコープ・非機能）
- `glossary.md`（現場用語の揺れを吸収）
- `schemas.md`（データ/APIの約束事）
- `tests.md`（受け入れ条件の固定）

運用:
- 変更は小さく、履歴が追える単位で更新
- 重要な前提（タイムゾーン/休日/締め日など）は必ず明記
